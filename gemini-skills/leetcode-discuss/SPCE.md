## Golden mindset of AI skills:
Always: To improve LLM productivity, human-guide the skill creation, but prompt to the LLM that skills are machine-actionable not human-redable.
Human is the guidance, and LLM speaks the mutual language with skills for reusablility
Think human as a manager of a team of agents, and the skill is the tool that the agents use to execute tasks. The skill should be designed in a way that it can be easily used by the agents without needing to understand the underlying implementation details. (or only a small fraction of the details)
The skill should also be designed to be reusable across different tasks and contexts, so that it can be easily adapted to new situations without needing to be rewritten from scratch.

## Expected skill args
/leetcode-discuss [problem name or id or link-to-problem] [question or topic to discuss] [accepted or not?]

## Context
1. I want to discuss a leetcoed problem, but in an interactive way, where I can ask questions and get answers about the problem, its solution, and related topics.

## Target
1. When the problem is accepted, you will ask me to provide my accepted code solution (most of the code will be providied with detailed self-analysis)
